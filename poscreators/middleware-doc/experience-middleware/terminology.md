# Terminology

The table below contains descriptions and explanations of terms commonly used in the context of the **Experience Middleware**.

These terms relate to customer-facing features such as digital receipts, in-store applications, and optional payment integrations.  
For fiscal and legal terminology, refer to the general Middleware terminology.

| Term | Description |
|------|-------------|
| Customer-Facing Device | A screen or terminal visible to the consumer during checkout, used to display digital receipts, QR codes, or interaction options. |
| Digital Receipt | A receipt generated after fiscalization and delivered to the consumer in a digital format (for example via QR code or HTTPS link), while remaining linked to the fiscal receipt. |
| Experience Flow | A combined sequence of checkout-related events that can include payment, fiscalization, receipt generation, and receipt handover. |
| Experience Middleware | Functional part of the fiskaltrust.Middleware that focuses on customer experience features such as digital receipt delivery, customer interaction, and optional payment handling. |
| HTTPS Receipt | A digitally accessible receipt delivered via a secure HTTPS link, typically accessed through a QR code displayed at checkout. |
| InStore App | A customer-facing application provided by fiskaltrust that displays receipts and receipt interaction options on a device at the point of sale. |
| Modular Feature | An optional Experience Middleware capability that can be enabled or disabled independently, such as digital receipts, InStore App usage, or payment integration. |
| Payment Provider (PSP) | A third-party service responsible for processing payment transactions, which can be accessed through the fiskaltrust payment integration. |
| Payment Transaction | The technical process of authorizing and completing a payment, which can be linked to a fiscalized receipt for traceability. |
| POS (Point of Sale) | The system used by merchants to process sales, initiate payments, and communicate with the fiskaltrust.Middleware. |
| QR Code | A machine-readable code displayed on a customer-facing device that allows consumers to access a digital receipt using a mobile device. |
| Receipt Acknowledgement | A logged confirmation that a consumer has received or accepted a receipt, triggered by user interaction or predefined flow logic. |
| Receipt Handover | The process of making a receipt available to the consumer after checkout, either digitally or in printed form. |
| Unified Payment Endpoint | A standardized interface provided by fiskaltrust to initiate and process payments while remaining independent of specific payment service providers. |
